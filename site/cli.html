CLI Reference

Every command the dippin toolchain provides — from authoring and formatting to analysis and testing. All commands work offline against workflow source files.

Global Flags

dippin [--format text|json] <command> [args]
FlagValuesDefaultDescription
--format text, json text Output format for diagnostics. json produces machine-readable arrays for CI and tooling.

Exit Codes

All commands use consistent exit codes.

CodeMeaning
0Success — no issues found, operation completed
1Error — validation failures, parse errors, check-mode drift
2Usage error — bad flags, missing arguments, unknown command

Authoring Commands

Authoring

parse

dippin parse <file>

Parse a workflow file and output the intermediate representation (IR) as JSON. Useful for debugging and tooling integration.

validate

dippin validate <file>

Run structural validation checks (DIP001–DIP009). These are the rules that must pass for a workflow to be executable.

lint

dippin lint <file>

Run all 31 diagnostic rules — structural validation plus semantic linting (DIP001–DIP009, DIP101–DIP122). Warnings don't affect exit code.

check

dippin check [--format json|text] <file>

Parse, validate, and lint in one shot. Defaults to JSON output. Designed for LLM tool-calling loops and CI pipelines.

fmt

dippin fmt [--check] [--write] <file>

Format a .dip file to canonical form. Use --check in CI, --write to format in place.

new

dippin new [--name <name>] [--write <file>] <template>

Generate a starter .dip file from a built-in template. Templates: minimal, parallel, conditional, review-loop, human-gate.

Export & Migration

Export

export-dot

dippin export-dot [--rankdir=LR|TB] [--prompts] <file>

Export a workflow to Graphviz DOT format for visualization. Pipe to dot -Tpng to render.

migrate

dippin migrate [--output <file>] <file.dot>

Convert a DOT file to .dip source format. Handles shape mapping, prompt unescaping, and condition prefixing.

validate-migration

dippin validate-migration <old.dot> <new.dip>

Check structural parity between a DOT file and a .dip file to verify migration correctness.

Analysis Commands

Analysis

simulate

dippin simulate [--scenario key=val] [--interactive] [--all-paths] <file>

Dry-run a workflow's execution graph without calling LLMs. Emits JSONL events for each node enter, exit, and edge traversal.

cost

dippin cost <file>

Estimate workflow execution cost based on model pricing tables. Breaks down per-node and total costs.

coverage

dippin coverage <file>

Analyze edge coverage and reachability. Reports which nodes and edges are exercised.

doctor

dippin doctor <file>

Health report card aggregating lint, coverage, and cost into a letter grade (A–F) with actionable suggestions.

test

dippin test [--verbose] <file.dip>

Run scenario tests from .test.json files. Injects context, simulates paths, checks assertions. Learn more →

In Action

check (JSON output) 
$ dippin check pipeline.dip
{
  "valid": true,
  "errors": 0,
  "warnings": 0,
  "diagnostics": [],
  "suggested_actions": []
}

$ dippin check broken.dip
{
  "valid": false,
  "errors": 1,
  "warnings": 2,
  "diagnostics": [
    {"code": "DIP003", "severity": "error", ...}
  ]
}
generate & validate 
$ dippin new --name MyPipeline review-loop --write my_pipeline.dip
wrote my_pipeline.dip

$ dippin validate my_pipeline.dip
validation passed

$ dippin export-dot my_pipeline.dip | dot -Tpng -o pipeline.png
# renders workflow graph as PNG
