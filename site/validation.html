Validation & Linting

31 diagnostic codes catch problems before runtime

Overview

Dippin provides two layers of static analysis. Structural validation (DIP001–DIP009) catches errors that must be fixed — a workflow with any of these cannot execute. Semantic linting (DIP101–DIP122) flags warnings: likely bugs or questionable patterns that should be reviewed but don't block execution.

Run dippin validate <file> for structural checks only, or dippin lint <file> for both.

Diagnostic Format

Diagnostics use a rustc-inspired format with code, location, and optional help text:

error[DIP003]: unknown node reference "InterpretX" in edge
  --> pipeline.dip:45:5
  = help: did you mean "Interpret"?

Each diagnostic includes:

FieldDescription
CodeUnique identifier (e.g., DIP003)
Severityerror, warning, info, or hint
MessageHuman-readable explanation
LocationFile, line, and column
HelpOptional suggestion for fixing
FixOptional replacement text

Use --format json for machine-readable output in CI.

Structural Errors (DIP001–DIP009)

These are hard errors. A workflow with any of these cannot be executed.

DIP001Start Node Missing
The workflow must declare a start: field pointing to an existing node.
error[DIP001]: start node does not exist
  --> pipeline.dip:1:1
  = help: add "start: <NodeID>" to the workflow header
DIP002Exit Node Missing
The workflow must declare an exit: field pointing to an existing node.
error[DIP002]: exit node does not exist
  --> pipeline.dip:1:1
  = help: add "exit: <NodeID>" to the workflow header
DIP003Unknown Node Reference in Edge
Every edge endpoint must reference an existing node. Uses Levenshtein distance to suggest corrections for typos.
error[DIP003]: unknown node reference "InterpretX" in edge
  --> pipeline.dip:45:5
  = help: did you mean "Interpret"?
DIP004Unreachable Node from Start
Every node must be reachable from the start node via BFS. Orphaned nodes are dead code.
error[DIP004]: node unreachable from start
  --> pipeline.dip:20:3
  = help: add an edge leading to this node, or remove it
DIP005Unconditional Cycle Detected
The graph must be a DAG. Restart edges (restart: true) are excluded from cycle detection.
error[DIP005]: unconditional cycle detected
  --> pipeline.dip:50:5
  = help: remove an edge in this cycle or mark it "restart: true"
DIP006Exit Node Has Outgoing Edges
The exit node is terminal — it must have zero outgoing edges.
error[DIP006]: exit node has outgoing edges
  --> pipeline.dip:55:5
  = help: remove outgoing edges from the exit node
DIP007Parallel/Fan-In Mismatch
Every parallel node must have a matching fan_in node with the same set of branch nodes.
error[DIP007]: parallel fan-out/fan-in mismatch
  --> pipeline.dip:15:3
  = help: add a matching fan_in node
DIP008Duplicate Node ID
Node IDs must be globally unique within a workflow.
error[DIP008]: duplicate node ID
  --> pipeline.dip:30:3
  = help: rename this node or remove the duplicate
DIP009Duplicate Edge
No two edges may have the same (from, to, condition) combination. Different conditions on the same pair are fine.
error[DIP009]: duplicate edge
  --> pipeline.dip:60:5
  = help: remove the duplicate edge

Semantic Warnings (DIP101–DIP122)

These are warnings that flag likely bugs. They don't block execution but should be reviewed. Showing the most important ones here.

DIP101Node Only Reachable via Conditional Edges
All incoming edges have conditions — if none match, this node is unreachable at runtime. Automatically suppressed when source conditions are exhaustive (e.g., success + fail).
warning[DIP101]: node "NextPhase" is only reachable through conditional edges
  --> pipeline.dip:25:3
  = help: add an unconditional edge, or verify conditions are exhaustive
DIP102Routing Node Missing Default Edge
A node with conditional outgoing edges but no unconditional fallback. Execution gets stuck if no condition matches. Also suppressed for exhaustive conditions.
warning[DIP102]: node "Gate" has conditional edges but no default
  --> pipeline.dip:35:3
  = help: add an unconditional edge as a fallback
DIP105No Success Path to Exit
Every path from start to exit goes through at least one conditional edge. If conditions don't match, execution may never reach the exit.
warning[DIP105]: no success path from start to exit
  --> pipeline.dip:1:1
DIP108Unknown Model/Provider
The model or provider string doesn't match any known LLM provider. Catches typos like claude-sonet-4 before they hit production.
warning[DIP108]: unknown model/provider combination
  --> pipeline.dip:15:5
DIP110Empty Prompt on Agent
An agent node has no prompt text. An agent without a prompt won't produce meaningful output.
warning[DIP110]: empty prompt on agent node
  --> pipeline.dip:12:3
DIP111Tool Without Timeout
A tool node defines a command but no timeout. Without a timeout, a hanging command blocks the pipeline indefinitely.
warning[DIP111]: tool command has no timeout
  --> pipeline.dip:35:3
DIP115Goal Gate Without Recovery Path
A node has goal_gate: true but no retry or fallback target. The pipeline has no way to recover if the gate fails.
warning[DIP115]: node "validate_tests" has goal_gate but no recovery
  --> pipeline.dip:18:3
  = help: add retry_target or fallback_target
DIP121Condition References Variable Not in Writes
An edge condition references a variable the source node doesn't declare in writes. Only fires when the source node has non-empty writes declarations.
warning[DIP121]: edge Gate → Pass: condition references "ctx.score"
  but node "Gate" does not declare it in writes
  --> pipeline.dip:45:5

Running Validation

Structural validation only

$ dippin validate pipeline.dip

Runs DIP001–DIP009. Exit code 0 if all pass, 1 if any errors.

Full lint (validation + semantic)

$ dippin lint pipeline.dip

Runs all 31 checks. Exit code 1 only for errors; warnings alone exit 0.

JSON output for CI

$ dippin --format json lint pipeline.dip

Emits diagnostics as a JSON array for machine consumption.
