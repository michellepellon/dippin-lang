# `.dipx` v1.1 — Bundle 1 (Cancellable Pack/Open + Source.Workflow ctx)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Thread `context.Context` through every CPU/IO loop that was previously uninterruptible (writeBundle, walkSourceTree, verifyAllHashes), and add `ctx` to the `Source.Workflow` signature so dirSource disk I/O is cancellable. This closes the asymmetry in spec § "Cancellation" (line 272) which Bundle 6 documented as a v1.1 contract.

**Architecture:** Five tasks. Task 1 changes the `Source.Workflow` signature (breaking — affects `Source` interface, `*Bundle`, `*dirSource`, and 6 test callers). Tasks 2-4 add `case <-ctx.Done()` at the top of each loop in `verifyAllHashes`, `walkSourceTree.visitNext`, and `writeBundle.writeAllPackedFiles`, with a cancellation regression test per loop. Task 5 is the CHANGELOG.

**Tech Stack:** Go 1.23. Pre-commit `just check` enforces gocyclo ≤ 5, gocognit ≤ 7.

**Roadmap:** `docs/superpowers/specs/2026-05-08-dipx-followups-roadmap-design.md` (Bundle 1).

**File map:**
- Modify: `dipx/source.go` (Source interface, dirSource.Workflow), `dipx/bundle.go` (Bundle.Workflow), test files updating callers — Task 1
- Modify: `dipx/helpers.go` (verifyAllHashes, walkSourceTree, writeAllPackedFiles), test files for cancellation — Tasks 2/3/4
- Modify: `dipx/dipx.go` (downstream wrappers may simplify) — Tasks 2/3/4
- Modify: `CHANGELOG.md` — Task 5

**Convention notes:**
- All 6 test callers of `.Workflow(...)` must be updated atomically with the signature change so the build stays green between commits.
- For loop-cancellation tests, use `context.WithCancel(context.Background())`; call `cancel()` mid-loop via a hook or detect via timing. The simplest pattern: pre-cancel a ctx and verify the function returns `ctx.Err()`.
